# Processo de Criacao de Landing Pages

Um processo documentado para criar Landing Pages com estrategia, copy, design, implementacao e auditoria. A intencao nao e gerar paginas parecidas entre si: e tomar decisoes explicitas, registrar suas referencias e validar o que foi construido antes da entrega.

## Comece por aqui

1. Leia a [arquitetura da brain](docs/arquitetura-da-brain.md) para entender o papel de cada camada.
2. Siga o [processo de criacao](docs/processo-de-criacao.md), sem pular os gates.
3. Use os [templates](#templates) para registrar as decisoes do projeto.
4. Consulte o [sistema de repertorio visual](docs/repertorio-visual.md) antes de desenhar.
5. Rode os [checklists](docs/checklists.md) antes de entregar.

## O principio

Uma Landing Page nao comeca no HTML. Ela comeca pela combinacao entre contexto comercial, oferta, publico, fonte de trafego, provas disponiveis e uma acao primaria clara.

O processo separa cinco coisas que costumam se misturar:

| Camada | Pergunta que responde | Resultado |
| --- | --- | --- |
| Estrategia | Para quem, qual problema e qual proximo passo? | Promessa, prova, objecoes e CTA |
| Especificacao | O que nao pode faltar? | Requisitos de estrutura, conversao e qualidade |
| Execucao | Em que ordem o trabalho acontece? | Fases com gates verificaveis |
| Repertorio visual | Como a pagina ganha identidade sem copiar referencias? | Decisoes visuais registradas por secao |
| Auditoria | Como saber se esta pronta? | Checklists objetivos |

## O fluxo em uma linha

`briefing -> estrategia -> wireframe -> copy -> mapa visual -> build -> auditoria -> entrega`

Cada seta representa uma decisao que precisa estar clara antes da proxima. Isso diminui retrabalho e evita que a interface vire uma colecao de blocos genericos.
